# نظرة عامة على المشروع

مشروع مستورد من GitHub لموقع مدارس تعليم قيادة على منصة **Blogger**:
- `template` — قالب Blogger (XML/Layout) الرئيسي للموقع.
- `pages/*.html` — صفحات ثابتة تُرفع كصفحات/عناصر داخل Blogger (لوحة تحكم admin، مراكز، أسئلة، مراجعات...).
- لا يوجد سيرفر/باك-إند تقليدي؛ البيانات (المستخدمون، الأسئلة، المراكز) مخزّنة في **Firebase Realtime Database** ويتم الوصول لها مباشرة من كود JavaScript داخل `pages/admin.html` والصفحات الأخرى.
- هذا المشروع لا يُشغَّل عبر workflow في Replit — الملفات تُنسخ/تُرفع يدويًا إلى Blogger والنطاق الفعلي `https://www.driverjo.online`.

## ميزة رفع صور الأسئلة (Cloudflare R2)

- سابقًا: حقل رفع صورة/GIF في `admin.html` (إدارة الأسئلة) كان يحوّل الملف إلى base64 ويخزّنه مباشرة في Firebase Realtime Database (`mediaUrl`).
- تم تغييره ليرفع الملف إلى **Cloudflare Worker** (`cloudflare-worker/upload-worker.js`) يخزّنه في **Cloudflare R2** ويُعيد رابطًا عامًا مستضافًا يُحفظ كـ `mediaUrl` بدل base64 الضخم.
- سكريبت الـ Worker وتعليمات النشر الكاملة موجودة في تعليق أعلى `cloudflare-worker/upload-worker.js`. القيم المستخدمة حاليًا مضمّنة في `pages/admin.html` (`UPLOAD_WORKER_URL`, `UPLOAD_KEY`).
- **ملاحظة أمنية معروفة:** `UPLOAD_KEY` مفتاح ثابت مضمّن في كود العميل (وليس سرًا حقيقيًا)؛ الحماية الوحيدة الإضافية هي تقييد CORS على نطاقات `ALLOWED_ORIGINS` داخل الـ Worker. هذا يطابق مستوى حماية لوحة التحكم الحالية (تحقق UID بسيط بلا مصادقة حقيقية) وهو مقبول لحجم الموقع الحالي.
- **بدائل تم استبعادها ولماذا:** Blogger API v3 لا يدعم استضافة صور خارج محرر الويب (تم اختبار ذلك فعليًا وفشل). Firebase Storage وCloudflare R2 كلاهما يتطلبان إضافة بطاقة/وسيلة دفع للحساب (حتى ضمن الحد المجاني)؛ Cloudinary لا يتطلب بطاقة لكن تم اختيار Cloudflare R2 بناءً على تفضيل المستخدم رغم شرط البطاقة.

## تفضيلات المستخدم
- لا تغييرات في هيكلة المشروع الحالية (يبقى Blogger + Firebase) إلا بطلب صريح.
