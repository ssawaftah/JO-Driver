Rebuild the admin dashboard's look and feel to match the SANAD app style
What & Why
The user wants pages/admin.html (the driverjo admin panel) rebuilt visually from scratch — "إعادة بناء كامل من الصفر بدون الاعتماد على أي تصميم حالي" — covering every section and form inside it, using the visual language shown in the reference screenshots in DESIGN/*.jpg (the Jordanian government "SANAD" mobile app: clean white surfaces, blue primary (~#1454F0-family) and green accent, heavily rounded cards, pill-shaped segmented tabs, soft shadows, generous spacing, bottom tab bar pattern). The user explicitly rejected keeping the current design as a base, even though the current CSS already has a SANAD-inspired blue palette — they want a genuinely new visual system, not incremental tweaks.

Scope — sections/forms to rebuild (all of pages/admin.html)
Login screen (Firebase email/password, allowlist check via admin/{uid} in Firebase RTDB)
Dashboard (stat cards, "أسئلة حسب القسم" and "أحدث طلبات الانتساب" summary panels)
Questions management: category grid → question list (search/filter by type: text/image/gif) → add/edit modal (category select, media type picker, URL vs. drag-and-drop image upload with client-side resize, dynamic options list with correct-answer selection)
Centers management: tab bar switching between "المراكز" and "طلبات الانتساب"; center list (search, filter by promoted/pinned); add/edit modal (name, phone, WhatsApp, Google Maps link with auto name-guessing, address, image upload, rating, weekly schedule builder with per-day open/closed toggle and time selects, description); request review/approve flow; promote-duration modal; delete confirmation modal
Users list (search, active filter)
Reviews grid (search, rating filter, delete)
Governorates & areas management (two-panel master/detail, add/edit/delete, edit-name modal)
Important constraints
This is a static single HTML file (pages/admin.html, ~3500 lines) with inline <style> and <script> — no build step, no framework. It is NOT part of the artifact/fullstack-app pipeline (no generated API client/hooks), so the standard "design subagent for fullstack apps" flow does not directly apply; treat it as a static-HTML mockup/rebuild instead.
All existing functionality must keep working after the visual rebuild: Firebase Auth + RTDB allowlist login, Cloudflare Worker REST API calls (CF.ref(...) wrapper hitting https://upload.idriverjo.workers.dev/api, documented in cloudflare-worker/README.md), image upload to the same Worker (X-Upload-Key header, client-side canvas resize ≤800px), Google Maps link parsing for center name guessing, and the weekly schedule builder logic.
Reference images: DESIGN/photo_5949542505953037875_y.jpg through ...888_y.jpg (14 screenshots of the SANAD app — login, help, map/stations, home cards, documents, services, account, attendance list).
Ask the user to clarify if a mobile-app-style bottom tab bar should literally replace the current desktop sidebar navigation, or if the SANAD look should be adapted to a desktop admin layout — this wasn't fully settled before scoping.
